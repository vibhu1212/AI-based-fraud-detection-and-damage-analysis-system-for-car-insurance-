## 2025-03-28 - Missing accessibility semantics on custom interactive React components
**Learning:** Found a systemic accessibility pattern specific to this app's UI components: interactive `div` elements used as custom buttons and file upload drop zones (e.g., `.upload-zone`, `.pipeline-node`, `.module-overview-card`) consistently lack keyboard event handlers (`onKeyDown`), `tabIndex`, `role="button"`, and visual focus states (`:focus-visible`). This completely breaks keyboard navigation for core features like uploading claim photos or toggling pipeline modules.
**Action:** When auditing or adding custom interactive elements (like drop zones or clickable cards), always explicitly add `role="button"`, `tabIndex={0}`, an `onKeyDown` handler that triggers on 'Enter' or 'Space', and a visible `:focus-visible` CSS outline to guarantee keyboard accessibility for screen reader and keyboard-only users.
