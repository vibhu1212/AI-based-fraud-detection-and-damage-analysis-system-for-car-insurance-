## 2025-03-05 - Fixed Path Traversal in StorageService
**Vulnerability:** The `StorageService` constructed file paths using string concatenation or basic `Path` operations (e.g., `self.storage_path / object_key`) without validating the resulting path. This allowed an attacker to supply object keys with directory traversal sequences (like `../../../etc/passwd` or absolute paths) to read, write, or delete arbitrary files outside the intended storage directory.
**Learning:** Even when using `pathlib`, appending an absolute path string or strings containing `..` to a base path can result in a path outside the base directory. `pathlib` resolves absolute additions by discarding the base path.
**Prevention:** Implemented an `_is_safe_path` check that strictly rejects strings starting with absolute path indicators or containing `..` segments. It additionally enforces canonical resolution using `.resolve().is_relative_to(base_dir)`. All storage operations (`upload_file`, `download_file`, `delete_file`, `file_exists`, `store_pdf`) now perform this validation before attempting file access.
