## 2024-05-15 - Path Traversal in Storage Service
**Vulnerability:** The `StorageService` in `backend/app/services/storage.py` concatenates unsanitized user-provided input (e.g., `object_key` or `filename`) directly to a base storage path (`self.storage_path / object_key`). This allows an attacker to supply a crafted path like `../../etc/passwd` to traverse outside the intended storage directory, potentially reading, writing, or deleting sensitive files on the server.
**Learning:** Even when abstracting storage operations (like S3-compatible interfaces over local filesystems), treating object keys as file paths without resolving and validating them against a base directory boundary is dangerous. Path joining alone does not sanitize `../` sequences.
**Prevention:** Always use `Path.resolve()` to get the canonical, absolute path of both the target file and the base directory. Then, use `Path.is_relative_to(base_dir)` to ensure the resolved file path strictly falls within the allowed base directory boundary before performing any file operations.
