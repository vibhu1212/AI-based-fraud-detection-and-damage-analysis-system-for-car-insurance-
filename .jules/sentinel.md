## 2025-03-09 - Path Traversal Vulnerability in Storage Service
**Vulnerability:** The `StorageService` allowed path traversal attacks through arbitrary `object_key` values. User-provided strings like `../../../../etc/passwd` could be used to read or write arbitrary files on the filesystem because `self.storage_path / object_key` directly concatenated the paths without resolving or validating them against the base storage directory.
**Learning:** Python's `pathlib.Path` behaves similarly to `os.path.join` where absolute paths or relative traversal sequences (`../`) in the right-hand operand can result in escaping the intended parent directory. Relying purely on basic path concatenation is insufficient for security when dealing with user-controlled input representing file paths.
**Prevention:** Always use `pathlib.Path.resolve()` to get the canonical, absolute representation of the path, and explicitly verify it remains within the intended base directory using `file_path.is_relative_to(base_path.resolve())` before performing any filesystem operations.
